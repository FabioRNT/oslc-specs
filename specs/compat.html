This document will address considerations with backwards compatibilty and co-existance considerations OSLC Core 2.0 when adopting 3.0. It is intended to cover the capabilities defined in OSLC Core 3.0.
This is an early working draft, working on outline and needs.

Introduction

The evolution of OSLC started from some simple documents that defined some XML and JSON formats and some interactions based on RESTful HTTP web services. These were labeled "OSLC 1.0". As a number of "1.0" specifications emerged, common patterns and reuse opportunities emerged. This then led to the creation of OSLC Core 2.0 [[OSLCCore2]]. From this, domain specifications were developed and other common specifications. The OSLC Core 2.0 evolved from 1.0 in that is strengthed its definition of the use of RESTful HTTP web services and adopted an existing standard-based approach to representing a data model that was flexible and had multiple representation formats, namely RDF [[rdf11-concepts]]. Now that OSLC Core 3.0 specifications are based on [[LINKED-DATA]] and namely the Linked Data Platform [[LDP]], there is a less-distruptive change from 2.0 to 3.0, as conceptually they are built on the same foundation: HTTP and RDF. Some of the underlying concepts and vocabularies may have changed in some areas, which this document will address how best to co-exist with 2.0 existing implemenations. Which in the end, will mostly be able just producing a few more RDF triples (or statements) in some of the responses or additional HTTP response headers.

OSLC-Core-Version

Deprecated. Technically, there is no to carry it forward in 3.0. Each capability should be able to accurately describe how to discover it using methods established by LDP such as HTTP response headers (Link relations, Accept-Post, ...).

Implementations that have previously tested for the value of this parameter, should ensure their logic properly accounts for values that may not be "2.0". As some may provide the value of "3.0", which shouldn't be seen as an error but instead as just an informative header.

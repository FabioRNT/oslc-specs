Introduction

Sometimes there are various resources and properties that are so commonly used or apply so broadly that it makes sense to have a place to define them: this is that place.

Common Properties and Resources

Common Properties

Changes to the OSLC Core Vocabulary MUST be approved by the OASIS OSLC Core TC. The OSLC Core Vocabulary is assigned the namespace URI of the http://open-services.net/ns/core#.

Domain TCs and other extensions MUST contribute their vocabulary terms in a namespace which is assigned to them as an authority.

OSLC Core, domain and other extensions SHOULD reuse existing vocabulary terms from stable vocabularies such as [[DC-TERMS]], RDF [[rdf11-concepts]], RDF Schema [[rdf-schema]], [[FOAF]], [[skos-reference]] and OSLC. New vocabulary terms should only be created when there is no clear existing choice available. See [[LDP]] similar clause on reuse.

Shapes

Shape: Discussion

Often times a series of comments is collected with a lifecycle resource, often referred to as a discussion. It is quite common across various types of resources such as project tasks, bug reports, requirements, assets and so on. This Discussion resource definition provides a minimal shape describes the needed properties.

Shape: Comment

Used in conjunction with to provide a minimal resource definition for a collection of comments.

Discovery

Not additional discovery needs. Vocabulary terms are discovered via published vocabulary documents at the OSLC Core namespace and shapes at advertised URLs.

RDF Vocabulary

Each specification may define new vocabulary terms. These should be defined with consideration for global use (not within a specific type of resource). Other domains and resource types will be encouraged to reuse these.

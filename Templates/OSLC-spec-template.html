        3.2 Shape

        For every resource that is defined, define a shape for it. You may have additional shapes for certain scenarios. Shapes must be published as separate Turtle files.

        • Name: xxx#ExampleClass1
        • Type URI: http://open-services.net/ns/xxx#ExampleClass1
        • Summary: Shape for resources of first example class
        • Description: A little more detail about this shape. Embedded HTML is ok.
        xxx#ExampleClass1 Properties
        Prefixed Name Occurs Read-only Value-type Representation Range Description
        dcterms:creator Zero-or-many unspecified AnyResource Either foaf:Agent, foaf:Person Creator or creators of resource. The link target is usually a foaf:Person or foaf:Agent, but could be any type.
        dcterms:title Zero-or-one unspecified XMLLiteral N/A Unspecified A title for this property. Embedded HTML is ok.
        oslc_xxx:color Zero-or-many unspecified Resource Reference oslc_xxx:Color Color of the subject

        3.3 Shape2

        Like first shape, included in same file, reusing some property definitions.

        • Name: xxx#ExampleClass2
        • Type URI: http://open-services.net/ns/xxx#ExampleClass2
        • Summary: Shape for resources of second example class
        xxx#ExampleClass2 Properties
        Prefixed Name Occurs Read-only Value-type Representation Range Description
        dcterms:creator Zero-or-many unspecified AnyResource Either foaf:Agent, foaf:Person Creator or creators of resource. The link target is usually a foaf:Person or foaf:Agent, but could be any type.
        dcterms:identifier Exactly-one true string N/A Unspecified A unique identifier for this resource.
        dcterms:title Zero-or-one unspecified XMLLiteral N/A Unspecified A title for this property. Embedded HTML is ok.

        3.4 Discovery

        This section details how OSLC clients can discover this capability. It should follow the model set by [LDP] and [OSLCCore3]. Depending on the capability, it could be defined using existing HTTP headers, HTTP OPTIONS response, content in responses such as a predicate that links a ldp:Container to some other kinds of resource (such as type URL or shape), and so on.
